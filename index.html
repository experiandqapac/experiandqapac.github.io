    Introduction

    Welcome to Experian Data Quality for Microsoft Dynamics CRM documentation page! You can use our API to verify addresses, phone numbers and emails. Contact us

    Deploy

    This section explains steps required to deploy the solution.

    Configuration

    This section explains configuration that can be made to customize the integration.

    There’s basically 3 main parts that can be configured for each deployment - services, data and touchpoints.

    {
  "debug": false,
  "targetPlatform": "MSCRM2016",
  "services": { },
  "data": { },
  "touchpoints": { }
}
    Parameter Description
    debug Flag indicating whether to log messages to the console of browser.
    targetPlatform The current supported platform is either MSCRM2016 or MSCRM2015.
    services Refer to service section.
    data Refer to data section.
    touchpoints Refer to Address, Email, Phone sections.

    Services

    This configures the REST APIs URL and credentials for all address, email and phone validation.
    {
  "services": {
    "address": {
      "url": "ADDRESS_SEARCH_REST_URL",
      "token": "ADDRESS_SEARCH_REST_TOKEN"
    },
    "phone": {
      "url": "PHONE_SEARCH_REST_URL",
      "token": "PHONE_SEARCH_REST_TOKEN"
    },
    "email": {
      "url": "EMAIL_SEARCH_REST_URL",
      "token": "EMAIL_SEARCH_REST_TOKEN"
    }
  },
  "data": { },
  "touchpoints": { }
}
    Parameter Description
    ADDRESS_SEARCH_REST_URL URL to the Capture API REST API.
    ADDRESS_SEARCH_REST_TOKEN Token for Capture API REST API (usually in the form of GUID).
    PHONE_SEARCH_REST_URL URL to the Capture API REST API.
    PHONE_SEARCH_REST_TOKEN Token for Capture API REST API (usually in the form of GUID).
    EMAIL_SEARCH_REST_URL URL to the Capture API REST API.
    EMAIL_SEARCH_REST_TOKEN Token for Capture API REST API (usually in the form of GUID).

    Data

    This configures the Data or Country datasets available for address, phone or email validation.
    {
  "services": { },
  "data": {
    "AUS": {
      "fullName": "Australia",
      "twoLetterCode": "au",
      "engine": "Intuitive",
      "layout": "MSDCRM2011",
      "placeholderText": "Search address here..."
    },
    "AUG": {
      "fullName": "Australia",
      "twoLetterCode": "au",
      "engine": "Intuitive",
      "layout": "MSDCRM2011",
      "placeholderText": "Search address here..."
    }
  },
  "touchpoints": { }
}
    Parameter Description
    fullName Name to be displayed in the country selector during address search.
    twoLetterCode Official two letter code of the country to be used in formatting phone numbers.
    engine Intuitive or Singleline.
    layout Layout of the search - if unsure MSCRM2011 can be used as default.
    placeholderText Text to be displayed before any search value is entered in text box.

    Touchpoint for Address Validate

    This configures specific search UI address, email or phone validations.
    {
  "services": { },
  "data": { },
  "touchpoints": [
    {
      "name": "WebResource_edq_vsm_addresscapture1",
      "type": "address",
      "addressStatus": "new_edq_vs_addresscapture1",
      "addressTime": "new_edq_vt_addresscapture1",
      "mappings": {
        "address1_line1": "street 1",
        "address1_line2": "street 2",
        "address1_line3": "street 3",
        "address1_city": "city",
        "address1_stateorprovince": "state/Province",
        "address1_postalcode": "ziP/Postal Code",
        "address1_country": {
          "type": "country"
        }
      },
      "countryMappings": {
        "AUG": "Australia",
        "AUS": "Australia",
        "NZD": "New Zealand"
      },
      "showCountry": false,
      "defaultCountry": "AUS"
    }
  ]
}
    Parameter Description
    name Name of the touchpoint to be created needs to be unique. The prefix “WebResource_edq_vsm_” is important for Phone and Email validate - because the remaining text ‘addresscapture1’ is treated as the field name to be hooked with email validation.
    type Touchpoint type.
    addressStatus Status field populated with Verified, Unverified or User Preferred automatically when search is completed.
    addressTime Time field populated with validation time when search is completed.
    mappings Mapping of field (MSCRM specific names) to layout returned.
    countryMappings Values to be populated to the country field based on country selected in UI.
    showCountry Show or hides the country selector (recommended to hide the country if only one country is expected all the time).
    defaultCountry Default country to be used in address search - must be valid country in data configuration.

    Touchpoint for Email Validate

    This configures specific search UI address, email or phone validations.
    {
  "services": { },
  "data": { },
  "touchpoints": [
    {
      "name": "emailaddress1",
      "type": "email",
      "verifiedLevels": [
        "Disposable",
        "Verified"
      ]
    }
  ]
}
    Parameter Description
    name Name of the touchpoint, also the name of the MSCRM field to be setup for validation.
    type Touchpoint type.
    verifiedLevels Levels returned from REST API that is considered verified. Leave this as default if unsure.

    Touchpoint for Address Validate

    This configures specific search UI address, email or phone validations.
    {
  "services": { },
  "data": { },
  "touchpoints": [
    {
      "name": "managerphone",
      "type": "phone",
      "countries": [
        "AU",
        "NZ"
      ],
      "defaultCountry": "AU",
      "verifiedLevels": [
        "Unreachable",
        "Absent",
        "Verified"
      ]
    }
  ]
}
    Parameter Description
    name Name of the touchpoint, also the name of the MSCRM field to be setup for validation.
    type Touchpoint type.
    countries Countries allowed in the phone field.
    defaultCountry Default country format used e.g When set to AU, 414535222 will be formatted to +61414535222 before being sent for validation.
    verifiedLevels Levels returned from REST API that is considered verified. Leave this as default if unsure.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
